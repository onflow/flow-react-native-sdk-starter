# AGENTS.md

This file guides AI coding agents (Claude Code, Codex, Cursor, Copilot, and others) working in this repository. Keep it concise — it is loaded into agent context automatically.

## Overview

Expo / React Native starter template for building Flow blockchain mobile apps. Wires up `@onflow/react-native-sdk` with WalletConnect for wallet auth, Cadence script reads, and transaction signing. Ships a single-screen demo (`app/index.tsx`) that queries FLOW balance and transfers FLOW on testnet/mainnet.

## Stack (verified via `package.json`)

- Expo SDK `~54.0.23` with Expo Router `~6.0.14`, new architecture enabled (`app.json`)
- React Native `0.81.5`, React `19.1.0`
- `@onflow/react-native-sdk` `^0.3.0`, `@onflow/fcl-react-native` `^1.22.0`
- `@walletconnect/sign-client` `^2.20.2`, `@walletconnect/react-native-compat` `^2.20.2`
- `@tanstack/react-query` `^5.67.3`, `viem` `^2.29.2`
- TypeScript `~5.9.2`, ESLint `^9.25.0` with `eslint-config-expo`

## Commands (from `package.json` scripts)

- `npm install` — install dependencies
- `npm start` — `expo start --clear` (dev server)
- `npm run ios` — `expo run:ios` (iOS build + Simulator)
- `npm run android` — `expo run:android` (Android build + Emulator)
- `npm run web` — `expo start --web`

No test script is defined. Cadence tests under `cadence/tests/` require the Flow CLI (`flow test`) and are not wired into npm scripts.

## Architecture

- `app/` — Expo Router file-based routes
  - `_layout.tsx` — root Stack, wraps children in `FlowProviderWrapper`
  - `index.tsx` — demo screen: `Connect` button, balance query, FLOW transfer form
  - `+native-intent.tsx` — intercepts `wc-redirect` deep links so WalletConnect returns don't trigger a navigation flash
- `components/flow-provider-wrapper.tsx` — **the actual Flow config lives here** (README mentions `config/flow.ts`, which does not exist). Wraps children in `FlowProvider` with hardcoded testnet `accessNodeUrl`, `discoveryWallet`, `discoveryAuthnEndpoint`, `flowNetwork`, `appDetail*` fields, and `walletconnectProjectId`. To switch to mainnet, edit this file.
- `cadence/` — Cadence 1.0 sources used with the Flow CLI (not imported at runtime by the app; `app/index.tsx` inlines its own Cadence strings)
  - `contracts/Counter.cdc` — sample contract (`access(all)`), emits `CounterIncremented`/`CounterDecremented`
  - `scripts/GetCounter.cdc`, `transactions/IncrementCounter.cdc`, `tests/Counter_test.cdc`
- `flow.json` — Flow CLI config. Single contract (`Counter`) with `testing` alias `0000000000000007`; networks for emulator / testnet / mainnet / testing; `emulator-account` only
- `assets/` — `app-icon.png`, `flow.png`
- `app.json` — Expo config. Scheme `flowreactnativesdkstarter`, `newArchEnabled: true`, `expo-router` plugin, `typedRoutes` experiment enabled
- `metro.config.js` — default Expo Metro config, no customisation
- `eslint.config.js` — flat config extending `eslint-config-expo/flat`
- `tsconfig.json` — extends `expo/tsconfig.base`, `strict: true`, path alias `@/*` → `./*`

## Conventions and Gotchas

- **Flow config is in `components/flow-provider-wrapper.tsx`, not `config/flow.ts`.** The README refers to a `config/flow.ts` file that does not exist in the repo. Network, `appDetail*`, and `walletconnectProjectId` all live in `flow-provider-wrapper.tsx`.
- **Hardcoded `walletconnectProjectId`** in `flow-provider-wrapper.tsx` — replace with your own WalletConnect Cloud project ID before shipping.
- **WalletConnect deep link**: `app/+native-intent.tsx` returns `/` for any path containing `wc-redirect`. Do not remove — without it the app shows a navigation flash on wallet return. See https://docs.expo.dev/router/advanced/native-intent/.
- **Contract addresses per network** are inlined in `app/index.tsx` (`CONTRACT_ADDRESSES` map for `FungibleToken` / `FlowToken` on testnet and mainnet). Cadence script/transaction strings are inlined in the same file — they are not read from `cadence/`.
- **Path alias**: import app-local modules as `@/...` (e.g. `@/components/flow-provider-wrapper`, `@/assets/flow.png`).
- **New Architecture is on** (`app.json` `newArchEnabled: true`); any native module added must support Fabric / TurboModules.
- **Typed routes** are enabled (`app.json` experiments); new routes will be reflected in generated types after `expo start`.
